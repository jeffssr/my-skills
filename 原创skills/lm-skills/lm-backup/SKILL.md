# lm-backup — 版本备份与恢复

为需求文档提供轻量版本控制。修改前备份，修改异常时可恢复。

---

## 触发

| 触发 | 不触发 |
|------|--------|
| 用户说"备份""先存一下" | 首次创建文件（无上一版本） |
| 用户说"改错了""退回去""回退""还原" → 直接恢复 | |
| 即将修改已有需求文档，**必须**先备份 | |
| 其他 skill 的备份门禁触发 | |

---

## 备份

### 命令

```bash
mkdir -p <需求目录>/.backup
```

对每个待备份文件：

```bash
# 确定序号：已有备份数 + 1
ls <需求目录>/.backup/<文件名>.r*.bak 2>/dev/null | wc -l  # 结果 +1 = 新序号
cp <文件> <需求目录>/.backup/<文件名>.r<序号>.bak
```

### 规则

- 备份位置：`<需求目录>/.backup/<文件名>.r<序号>.bak`
- 每次备份产生独立版本：同一文件改 N 次 = N 个备份（r1, r2, r3...），序号越大越新
- 首次创建的文件不需要备份
- 备份失败 → 停止修改，报告用户，禁止覆写
- 禁止假设"已经备份过了"——每次覆写前必须先备份，备份成功后才可继续修改

### 备份确认

备份完成后告知用户：

```
已备份：.backup/<文件名>.r<序号>.bak
当前 <文件名> 共有 <N> 个备份版本（r1~r<N>）
```

---

## 恢复

### 列出版本

```bash
ls <需求目录>/.backup/ | sort
```

### 恢复操作

```bash
cp <需求目录>/.backup/<文件名>.r<目标序号>.bak <原文件名>
```

### 恢复流程

1. `ls <需求目录>/.backup/ | sort` 列出可用版本
2. 告知用户各版本含义（r1=第1次修改前，r2=第2次修改前...）
3. 用户选择目标序号
4. `cp .backup/<文件名>.r<序号>.bak <原文件名>`
5. 确认恢复成功
