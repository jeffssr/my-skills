# 质疑过滤规则

审查员提出的质疑不等于整改指令。主代理必须用滤网逐条过滤。

## 滤网

| 序号 | 滤网 | 判定 | 处理 |
|------|------|------|------|
| 1 | 用户是否明确要求过？ | 是 → 保留 | |
| 2 | 是否有 spec/context/context.md 中的事实支持？ | 是 → 保留 | |
| 3 | 是否在当前产物的范围内？ | 是 → 保留 | |
| 4 | 类别是否为"建议优化"？ | 是 → **驳回** | 除非用户明确要求审查措辞/风格 |
| - | 以上均不满足 | 驳回 | 记录驳回理由，不整改 |

## 注意

- 审查员角色 = 质疑者，角色 ≠ 用户。审查意见 ≠ 用户需求
- 2 个审查员的质疑合并后统一过滤，不做"谁说的算"判断
- 驳回的质疑不整改，但记录在过滤报告中以备审计
- "建议优化"类质疑默认驳回，除非用户在审查决策时明确要求包含风格审查
