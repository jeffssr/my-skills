# 审查报告格式

## 审查员输出格式

文件：`<需求目录>/review-r{n}-{1|2}.md`

```markdown
# 审查报告 - 第 N 轮 - 审查员 {1|2}

## 审查维度

- [调用方提供的维度列表]

## 质疑项

| # | 类别 | 位置 | 问题描述 | 违反规则 | 建议修改 |
|---|------|------|---------|---------|---------|
| 1 | | | | | |

## 总结

- 质疑项总数：N
- 按类别分布：...
```

### 字段要求

| 字段 | 是否必填 | 要求 |
|------|---------|------|
| 序号 | 是 | 唯一编号 |
| 类别 | 是 | 见下方类别枚举 |
| 位置 | 是 | **必须定位到具体段落**：第N节第M段 / 第K行 / 引用原文片段。不接受"第3节有问题"这种模糊定位 |
| 问题描述 | 是 | 违反了什么 / 遗漏了什么 |
| 违反规则 | 是 | 引用具体规则原文或规则编号。不能自造规则 |
| 建议修改 | 是 | **具体的文本替换方案**，不接受"建议改一下""建议优化措辞"等空泛建议 |

### 类别枚举

| 类别 | 含义 | 过滤权重 |
|------|------|---------|
| 事实错误 | 与 spec/context 不一致或事实有误 | 高（默认保留） |
| 规则违反 | 违反写作约束/模板/规范 | 高（默认保留） |
| 遗漏 | 应有但缺失的内容 | 高（默认保留） |
| 逻辑矛盾 | 前后矛盾或逻辑断裂 | 高（默认保留） |
| 建议优化 | 措辞/风格/可读性优化建议 | **低（默认驳回）** |

**"建议优化"类别处理**：除非用户明确要求审查风格/措辞，否则过滤阶段自动驳回，不进入整改清单。

---

## 过滤报告格式

文件：`<需求目录>/review-filter-r{n}.md`

```markdown
# 质疑过滤报告 - 第 N 轮

## 保留项

| # | 原质疑# | 类别 | 位置 | 问题描述 | 通过滤网 | 整改方向 |
|---|---------|------|------|---------|---------|---------|

## 驳回项

| # | 原质疑# | 类别 | 位置 | 驳回理由 |
|---|---------|------|------|---------|

## 统计

- 保留：N 项
- 驳回：M 项
- 其中"建议优化"自动驳回：K 项
```
