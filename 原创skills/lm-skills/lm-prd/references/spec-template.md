# spec.md 模板

纯结构模板。写作规范见 [writing-rules.md](writing-rules.md)，通用约束见 `../lm-shared/constraints.md`。

---

## 文档结构

    # <需求标题>
    ├─ 一、综述
    │   ├─ 1 现状 & 问题
    │   ├─ 2 核心业务流程（可选）
    │   └─ 3 需求 List
    ├─ 二、平台端
    │   ├─ 1 ...
    │   └─ 2 ...
    ├─ 三、Portal 端
    │   └─ 1 ...
    └─ 四、邮件
        └─ 1 ...

端名用 `## 一、二、三、四...`，需求点用 `### 1/2/3...`（每端重新编号），子节用 `#### 1.1/1.2...`，更深子节用 `##### 1.1.1/1.2.1...`。

## 需求点

端名用 `## 一、二、...`，需求点用 `### 1/2/3...`（每端重新编号）。通用顺序：**编号 标题 → 位置 → 触发场景 → 原型 → 核心逻辑**。位置和触发场景分两行。

    ## 二、平台端

    **核心逻辑**：

## 补充说明

- **修改已有功能**：核心逻辑直接写修改后完整行为，不写 Before/After。原型标注写"需要（在原有基础上修改：…）"，除非用户明确要求重画。详见 [writing-rules.md](writing-rules.md)。
- **通知类**：格式详见 [writing-rules.md](writing-rules.md) 通知写法节。
- **多状态变体**：差异大（布局不同、字段差异 > 3 个、流程不同）→ 按变体逐个阐述；差异小（仅局部文案变化、个别字段显隐、标记状态变化）→ 整体说明，指出变化点。原型中用 state-label-inner 区分变体。详见 [writing-rules.md](writing-rules.md)。
