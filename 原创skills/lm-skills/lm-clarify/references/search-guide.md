## 子代理 prompt 模板

派子代理时，将以下 `<占位符>` 替换后发送：

```markdown
你是代码搜索专家。搜索 LinkMed 代码库，只读不改。搜索维度：<维度名>。

## 用户需求
<用户原始需求描述>

## 搜索方法
<从"搜索维度"表中复制对应维度的搜索方法>

## 语义扩展关键词
先基于用户需求做语义扩展（10-15 变体）：
- 中英双向映射
- 同义词/近义词
- 命名变体（camelCase/snake_case/PascalCase）
- 注释和文档关键词
- 字段名/枚举值
- 关联概念

## 搜索步骤
1. 用扩展关键词逐批 grep 搜索
2. 命中后 Read 完整文件（不只看片段）
3. 从命中内容发现新关键词追加搜索
4. 代码不在本仓库 → 搜 API 网关配置、服务发现、环境变量、文档找端点

## 输出格式

### 命中文件清单
| 文件路径 | 命中关键词 | 作用描述 |

### 关键发现
- <发现1>

### 新关键词建议
- <新关键词1>

### 未覆盖区域
- <未搜到的方向>
```

## 5 个搜索维度

| # | 维度 | 子代理标识 | 方法 |
|---|------|-----------|------|
| 1 | 本身要改的功能 | `search-r<n>-core` | 语义扩展搜索 → Read 完整文件 |
| 2 | 后端逻辑 | `search-r<n>-backend` | grep 语义搜索 Service/DAO/Entity/Controller |
| 3 | 关联影响排查 | `search-r<n>-impact` | 改了A→B/C/D受影响吗？列表筛选？导出？账单？日历？排班？日志？权限？ |
| 4 | 通知 | `search-r<n>-notify` | 读 `notifications/sms.md` + `notifications/email.md`，再 grep 代码 |
| 5 | 逐端逐页排查 | `search-r<n>-pages` | 对每个业务端逐个页面/抽屉/弹窗排查是否受影响 |

## 语义扩展搜索

用户说的词 ≠ 代码里写的词。每次搜索先做语义扩展：

1. **中英双向映射**："半天" → `halfDay`, `half_day`, `AM`, `PM`
2. **同义词/近义词**："请假" → `leave`, `absence`, `off`, `dayoff`
3. **命名变体**：`camelCase` / `snake_case` / `PascalCase` 全覆盖
4. **注释和文档**：中文注释、英文注释、Javadoc/TSDoc
5. **字段名/枚举值**：数据库字段、DTO、API 参数名
6. **关联概念**："半天"可能涉及 → `shift`, `schedule`, `timeslot`, `duration`

## 搜索流程

1. 列 10-15 个扩展关键词 → 逐批 grep
2. 命中后 Read 完整文件（不只看片段）
3. 从命中内容发现新关键词追加搜索
4. 代码不在本仓库 → 搜 API 网关配置、服务发现、环境变量、文档找端点
5. 换轮前总计已搜索关键词
6. 独立维度可派子代理并行搜索
